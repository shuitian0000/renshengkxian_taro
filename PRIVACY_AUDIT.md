# 微信小程序隐私审核优化总结

## 一、隐私相关问题检查结果

### 1. 已发现的隐私相关功能
- ✅ **相机权限**：用于拍摄面相照片（可选功能）
- ✅ **相册权限**：用于选择面相照片（可选功能）
- ✅ **用户信息**：微信昵称和头像（仅登录时获取）
- ✅ **命理信息**：姓名、性别、出生日期、出生时辰、出生地区

### 2. 不涉及的隐私功能
- ❌ 位置信息（不使用）
- ❌ 通讯录（不使用）
- ❌ 麦克风（不使用）
- ❌ 蓝牙（不使用）
- ❌ 运动数据（不使用）

## 二、已完成的优化措施

### 1. app.config.ts配置优化
✅ 添加了`permission`字段，说明相机和相册的使用目的：
```typescript
permission: {
  'scope.camera': {
    desc: '用于拍摄面相照片，辅助命理分析'
  },
  'scope.album': {
    desc: '用于选择面相照片，辅助命理分析'
  }
}
```

### 2. requiredPrivateInfos字段说明
✅ **不需要设置此字段**，因为：
- `requiredPrivateInfos`只能包含以下值：
  - chooseAddress（选择地址）
  - chooseLocation（选择位置）
  - choosePoi（选择POI）
  - getFuzzyLocation（获取模糊位置）
  - getLocation（获取精确位置）
  - onLocationChange（监听位置变化）
  - startLocationUpdate（开始位置更新）
  - startLocationUpdateBackground（后台位置更新）
- 我们的小程序不使用任何位置相关功能
- 相机和相册权限通过`permission`字段声明即可

### 3. 用户提示优化
✅ **首页明确标注**：
- 面相照片标题旁显示"（可选）"
- 页面底部提示"* 面相照片为可选项，不影响命理分析"

✅ **FaceUpload组件说明**：
- "上传正面免冠照片，面部、头部、耳朵清晰可见，可提升分析准确度"
- 明确告知用户照片用途和要求

### 4. 隐私保护指引文档
✅ 创建了`PRIVACY.md`文档，包含：
- 隐私信息收集与使用说明
- 权限申请时机说明
- 数据安全保障措施
- 用户权利说明
- 联系方式

## 三、符合微信审核规范的要点

### 1. 权限申请规范
✅ **相机和相册权限**：
- 在用户主动点击"选择照片"时才申请
- 有明确的用途说明
- 用户可以拒绝，不影响核心功能

✅ **用户信息授权**：
- 在用户主动点击"登录"时才申请
- 未登录用户可以正常使用命理分析功能
- 仅用于保存报告和查看历史

### 2. 隐私信息使用规范
✅ **最小化原则**：
- 只收集必要的信息
- 面相照片为可选项
- 不收集位置、通讯录等无关信息

✅ **透明化原则**：
- 在app.config.ts中明确说明权限用途
- 在UI上明确标注可选项
- 提供详细的隐私保护指引

✅ **用户控制原则**：
- 用户可以选择不提供面相照片
- 用户可以选择不登录
- 用户可以随时删除数据

### 3. 数据安全规范
✅ **传输安全**：
- 使用HTTPS加密传输
- 使用Supabase云服务，符合安全标准

✅ **存储安全**：
- 敏感信息加密存储
- 用户数据隔离，只能访问自己的数据

✅ **访问控制**：
- 实现了Row Level Security（RLS）
- 用户只能访问自己的数据

## 四、审核提交建议

### 1. 隐私保护指引
在微信小程序后台"设置 > 基本设置 > 服务内容声明"中：
- 上传隐私保护指引（PRIVACY.md内容）
- 说明相机和相册的使用场景和目的
- 说明用户信息的收集和使用方式

### 2. 用户隐私保护指引
在微信小程序后台"设置 > 基本设置 > 用户隐私保护指引"中：
- 填写隐私政策链接（如有）
- 说明收集的信息类型和用途
- 说明用户权利和联系方式

### 3. 接口权限申请
在微信小程序后台"开发 > 开发管理 > 接口设置"中：
- 申请"相机"权限，说明用途："用于拍摄面相照片，辅助命理分析"
- 申请"相册"权限，说明用途："用于选择面相照片，辅助命理分析"
- 申请"获取用户信息"权限，说明用途："用于用户身份识别和报告保存"

### 4. 代码审核说明
在提交审核时的"备注"中说明：
```
本小程序涉及以下隐私权限：
1. 相机权限：用于拍摄面相照片，辅助命理分析（可选功能）
2. 相册权限：用于选择面相照片，辅助命理分析（可选功能）
3. 用户信息：用于用户身份识别和报告保存（仅登录时获取）

所有权限均在用户主动操作时申请，有明确的用途说明，用户可以拒绝授权。
未授权不影响核心功能使用。
```

## 五、注意事项

### 1. 不要设置requiredPrivateInfos
- 该字段只用于位置相关API
- 我们的小程序不使用位置信息
- 不需要设置此字段

### 2. permission字段的作用
- 用于说明权限的使用目的
- 会在用户授权时显示
- 必须简洁明确，不超过30个字

### 3. 隐私保护指引的重要性
- 微信要求所有小程序都要有隐私保护指引
- 必须在小程序内或官网提供
- 内容必须真实、完整、准确
